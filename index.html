What is manifest.json?

Every Minecraft Bedrock Edition resource pack and behavior pack requires a manifest.json file in its root directory. This file tells the game engine what the pack is, what it does, and gives it a unique identity through UUID v4 identifiers.

Why do you need two UUIDs?

Bedrock manifest.json requires two separate UUIDs: one for the pack header (identifies the pack itself) and one for the module (identifies the pack's content type — resources or data). Both must be unique UUID v4 values.

Who is this tool for?

  • PocketMine-MP developers creating custom resource packs for their servers
  • Bedrock Edition addon creators building texture packs, skin packs, and behavior packs
  • MCPE modders who need valid manifests for Pocket Edition
  • Anyone working with Minecraft Bedrock resource packs

How to use the generated manifest.json

  • Place manifest.json in the root folder of your resource pack
  • Add a pack_icon.png (256×256 recommended) next to it
  • Zip the folder and rename to .mcpack for easy installation
  • For PocketMine-MP servers, place the pack in the resource_packs directory

Supported Versions

This generator supports both format_version 1 (legacy) and format_version 2 (1.16.100+). Format version 2 is recommended for all modern Bedrock Edition content, including the latest PocketMine-MP versions.
